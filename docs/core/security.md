# JustMemo 私密 Memo 规则

> 最后更新：2026-04-19
> 状态：产品规则已确认，代码实现待对齐

## 1. 核心定义

JustMemo 中的 Memo 只有两种可见性：

- `public`：公开内容，任何用户都可以直接查看正文。
- `private`：私密内容，正文默认不展示，按条目密码保护。

`private` 的语义必须保持单一：

- 不是“登录用户可见”
- 不是“平台管理员可见”
- 不是“仅自己可见”
- 而是“作者本人可见，其他人需按该条 Memo 单独解锁后可见”

## 2. 权限模型

系统中必须区分两类权限：

- `内容所有权 (owner)`：这条 Memo 的作者是谁。
- `平台管理权限 (platform admin)`：站点运维能力，如用户管理、系统维护。

私密 Memo 的阅读权限只由 `owner` 和“是否已解锁该条 Memo”决定，不能由 `platform admin` 决定。

### 2.1 阅读权限

- 作者本人可以直接查看自己写的私密 Memo。
- 非作者即使已登录，也不能直接查看私密 Memo。
- 非作者只有输入该条私密 Memo 的正确密码后，才能查看该条正文。
- 平台管理员不自动拥有查看他人私密 Memo 的能力。

### 2.2 管理权限

- 创建 Memo：作者本人。
- 编辑 Memo：作者本人。
- 删除 Memo：作者本人。
- 修改私密状态、密码、提示：作者本人。
- 备份导出：作者本人只能导出自己的 Memo。

## 3. 私密 Memo 的基础行为

### 3.1 创建与存储

- 每条私密 Memo 必须有独立的密码和提示。
- 密码只允许以哈希形式存储，不能明文落库。
- 私密属性和密码绑定在单条 Memo 上，不能共用全局口令状态。

### 3.2 解锁

- 解锁必须是按 `memo_id` 粒度生效。
- 解锁只对当前页面生命周期生效。
- 为了安全，关闭页面后解锁状态失效。
- 按当前产品约定，刷新页面后也应失效，需要重新输入密码。

### 3.3 会话约束

私密 Memo 不应使用持久化解锁状态：

- 不写入长期 Cookie
- 不写入 `localStorage`
- 不应形成“解锁一条，整站都解锁”的行为

## 4. 各功能表现

### 4.1 主列表 / Feed

- 作者本人：直接看到正文。
- 非作者未解锁：只显示锁定态占位。
- 非作者已解锁该条：仅该条显示正文。

### 4.2 搜索

- 未解锁的私密 Memo 不参与搜索结果。
- 作者本人可搜索到自己的私密 Memo。
- 非作者只有在已解锁该条 Memo 后，搜索结果中才允许出现该条 Memo。

### 4.3 地图

- 只要 Memo 有坐标，就可以在地图上显示标记。
- 私密 Memo 的地图标记可以存在，但点击后默认仍是锁定态。
- 作者本人点击标记可以直接看到内容。
- 非作者点击私密标记后，必须先解锁，解锁成功后才显示具体内容。

### 4.4 时间轴

- 时间轴当前不做特殊增强。
- 右侧跳转到某一天后，当天列表里的私密 Memo 仍按主列表规则处理。
- 未解锁时仍保持锁定态，不显示正文。

### 4.5 标签

- 私密 Memo 不参与标签统计。
- 私密 Memo 不应进入公共标签聚合和计数结果。

### 4.6 分享

- 私密 Memo 不允许生成分享图。
- 私密 Memo 不允许提供对外分享能力。

### 4.7 备份

- 备份只能由作者本人执行。
- 备份范围仅限作者自己的 Memo。
- 平台管理员不应具备导出其他用户私密 Memo 的能力。

## 5. 默认安全原则

凡是衍生功能，都必须继承同一条判断规则：

- 作者本人可见
- 已解锁该条 Memo 的用户可见
- 其他人不可见或仅见锁定占位

如果某个功能暂时无法完整支持这套规则，应优先选择“隐藏”而不是“先暴露”。

## 6. 当前实现与目标规则的差异

以下差异已经确认，后续代码改造必须逐项收敛：

1. 当前实现错误地把“已登录用户”当成了私密内容的天然可见者。
2. 当前实现中部分数据库策略仍以 `admin` / `authenticated` 为中心，而不是以 `owner` 为中心。
3. 当前解锁状态不是严格按 `memo_id` 粒度设计，且存在全局口令授权倾向。
4. 搜索、标签、地图、统计等衍生功能还没有完全统一到“作者 / 单条解锁”的权限模型。
5. 私密 Memo 的分享与备份边界需要进一步按“作者本人”重构。

## 7. 后续改造原则

后续代码设计必须围绕以下三点展开：

1. 引入明确的 `owner_id` 语义，让内容所有权成为私密权限判断的基础。
2. 将“平台管理能力”和“私密内容阅读能力”彻底解耦。
3. 将解锁状态收敛为“当前页面内的已解锁 Memo 集合”，而不是全局密码状态。
