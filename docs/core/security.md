# JustMemo 私密 Memo 规则

> 最后更新：2026-04-19
> 状态：已实现并与当前代码同步

## 1. 作用

这份文档定义私密 Memo 的产品规则与工程边界。

它回答的是：

- 谁可以看私密 Memo
- 解锁如何生效
- 搜索、地图、时间轴、标签、备份与分享应如何继承私密规则

## 2. 核心定义

JustMemo 中的 Memo 只有两种可见性：

- `public`：公开内容，任何用户都可以直接查看正文。
- `private`：私密内容，正文默认不展示，按条目密码保护。

`private` 的语义必须保持单一：

- 不是“登录用户可见”
- 不是“平台管理员可见”
- 不是“仅自己可见”
- 而是“作者本人可见，其他人需按该条 Memo 单独解锁后可见”

## 3. 权限模型

系统中必须区分两类权限：

- `内容所有权 (owner)`：这条 Memo 的作者是谁。
- `平台管理权限 (platform admin)`：站点运维能力，如用户管理、系统维护。

私密 Memo 的阅读权限只由 `owner` 和“是否已解锁该条 Memo”决定，不能由 `platform admin` 决定。

### 2.1 阅读权限

- 作者本人可以直接查看自己写的私密 Memo。
- 非作者即使已登录，也不能直接查看私密 Memo。
- 非作者只有输入该条私密 Memo 的正确密码后，才能查看该条正文。
- 平台管理员不自动拥有查看他人私密 Memo 的能力。

### 2.2 管理权限

- 创建 Memo：作者本人。
- 编辑 Memo：作者本人。
- 删除 Memo：作者本人。
- 修改私密状态、密码、提示：作者本人。
- 备份导出：作者本人只能导出自己的 Memo。

## 4. 私密 Memo 的基础行为

### 3.1 创建与存储

- 每条私密 Memo 必须有独立的密码和提示。
- 密码只允许以哈希形式存储，不能明文落库。
- 私密属性和密码绑定在单条 Memo 上，不能共用全局口令状态。

### 3.2 解锁

- 解锁必须是按 `memo_id` 粒度生效。
- 解锁只对当前页面生命周期生效。
- 为了安全，关闭页面后解锁状态失效。
- 按当前产品约定，刷新页面后也应失效，需要重新输入密码。

### 3.3 会话约束

私密 Memo 不应使用持久化解锁状态：

- 不写入长期 Cookie
- 不写入 `localStorage`
- 不应形成“解锁一条，整站都解锁”的行为

## 5. 各功能表现

### 4.1 主列表 / Feed

- 作者本人：直接看到正文。
- 非作者未解锁：只显示锁定态占位。
- 非作者已解锁该条：仅该条显示正文。

### 4.2 搜索

- 未解锁的私密 Memo 不参与搜索结果。
- 作者本人可搜索到自己的私密 Memo。
- 非作者只有在已解锁该条 Memo 后，搜索结果中才允许出现该条 Memo。

### 4.3 地图

- 只要 Memo 有坐标，就可以在地图上显示标记。
- 私密 Memo 的地图标记可以存在，但点击后默认仍是锁定态。
- 作者本人点击标记可以直接看到内容。
- 非作者点击私密标记后，必须先解锁，解锁成功后才显示具体内容。

### 4.4 时间轴

- 时间轴当前不做特殊增强。
- 右侧跳转到某一天后，当天列表里的私密 Memo 仍按主列表规则处理。
- 未解锁时仍保持锁定态，不显示正文。

### 4.5 标签

- 私密 Memo 不参与标签统计。
- 私密 Memo 不应进入公共标签聚合和计数结果。

### 4.6 分享

- 私密 Memo 不允许生成分享图。
- 私密 Memo 不允许提供对外分享能力。

### 4.7 备份

- 备份只能由作者本人执行。
- 备份范围仅限作者自己的 Memo。
- 平台管理员不应具备导出其他用户私密 Memo 的能力。

## 6. 默认安全原则

凡是衍生功能，都必须继承同一条判断规则：

- 作者本人可见
- 已解锁该条 Memo 的用户可见
- 其他人不可见或仅见锁定占位

如果某个功能暂时无法完整支持这套规则，应优先选择“隐藏”而不是“先暴露”。

## 7. 当前实现落地

当前代码已按上述规则完成收敛，关键实现如下：

1. `memos.owner_id` 已成为私密权限判断的基础字段。
2. 数据库 RLS 已切换为“作者可读写自己的 Memo”，平台管理员不再天然拥有他人私密内容读取权。
3. `search_memos_secure` 已改为接收 `unlocked_ids UUID[]`，按条目判断解锁，而不是接受全局口令。
4. 客户端使用 `UnlockedMemosContext` 在内存中维护当前页面已解锁 Memo 集合，不写入 Cookie 或 `localStorage`。
5. 搜索、主列表、地图、时间轴、标签统计、备份导出和分享能力都已切换到“作者 / 单条解锁”模型。

## 8. 实现约束

后续涉及私密 Memo 的代码改动，必须继续满足以下约束：

1. 新功能若需要判断可见性，优先复用统一的可见性辅助，而不是单独发明一套权限分支。
2. 凡是会暴露正文、标签、字数、分享内容的地方，都要区分“作者可见”“已解锁可见”“其他人仅锁定占位”三种状态。
3. 任何新增缓存、持久化或跨页面状态同步方案，都不能把私密解锁状态做成长期凭证。
