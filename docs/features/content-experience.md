# JustMemo 内容体验

> 最后更新：2026-04-26
> 状态：已实现

## 1. 作用

这份文档描述的是正文内容在阅读与编辑过程中的增强体验，包括解析规则、链接预览、邮件地址交互和子页面缓存预热。

## 2. 内容解析

正文统一由内容解析层处理，解析优先级为：

1. Code Block
2. Location
3. Markdown Image
4. Mention / Ref
5. Tag
6. Email
7. Raw Image URL
8. Markup Link
9. Raw Link

其中需要注意：

- `Image` 在实现里分为 Markdown 图片和图片直链两类
- 除普通链接外，还支持 `🔗[title](url)` 这种标记链接语法，以及带显示模式的变体

这样可以避免一段文本被多个规则错误重复命中。

## 3. 链接预览

链接预览用于提升外链的上下文感知。

- 当正文中检测到标准 URL 时，会异步请求元数据。
- 展示内容包括标题、描述与站点标识。
- 展示方式尽量轻量，避免让预览卡片压过正文本身。

## 4. 邮件地址交互

邮件地址属于正文中的特殊可操作内容：

- 自动识别邮件地址
- 将其渲染为 `mailto:` 链接
- 悬浮时提供快捷复制能力

邮件复制按钮使用小尺寸图标，因此视觉规范上需要确保高分屏下的清晰度。

## 5. 缓存预热

当用户访问某些子页面时，例如：

- 画廊
- 地图

页面会优先检查现有 Memo 缓存：

- 若缓存为空，则主动拉取轻量索引
- 用一次预热为后续站内导航加速

## 6. 和其他模块的关系

- 图片解析会直接影响画廊体验。
- 坐标解析会直接影响地图功能。
- 邮件和链接增强属于正文阅读体验，不应写成独立业务页面逻辑。

## 7. 维护边界

- 新增新的正文语法时，应先判断它是否属于解析层扩展。
- 若属于，应优先更新这份文档，而不是把规则散落到各个功能页里。
