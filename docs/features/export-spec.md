# 导出与分享功能规格说明书 (Export & Share Specification)

## 1. 愿景与目标

将 JustBB 的导出系统从单纯的数据备份工具提升为一种“数字策展”体验。让用户能够以最具人文美感的形式分享思考碎片，并确保数据的安全与可迁移性。

---

## 2. 核心需求 (Requirements)

### 2.1 视觉海报分享 (Visual Poster)

- **多主题支持**：用户应能根据内容基调切换不同的海报主题。
- **品牌识别度**：导出的海报必须包含 JustBB/JustMemo 品牌水印、记录日期及溯源二维码。
- **高质量渲染**：导出图片需支持 2x 采样，确保高清显示。
- **排版稳定性**：即使在极端文本长度下，海报布局也应保持平衡。

### 2.2 数据备份 (Data Backup)

- **多格式支持**：提供 JSON (数据完整性) 和 Markdown (可读性) 两种选项。
- **本地化处理**：所有导出逻辑均在客户端完成。

---

## 3. 设计标准 (Design Standards)

- **人文美学**：遵循 `DESIGN.md` 的“Warm Neutral”色调。
- **排版细节**：使用衬线体（Serif）增强质感，行高与字间距需经过微调。
- **实时预览**：在保存前提供 1:1 的视觉预览。

---

## 4. 演进路线图 (Roadmap)

### 阶段一：海报美学升级 [进行中]

- **目标**：重构海报引擎，引入多主题切换 UI。
- **交付**：Classic, Zen, Midnight 三套主题。

### 阶段二：批量处理与长图 [待办]

- **目标**：支持多选 Memo 并合并生成长图或 PDF 报告。

### 阶段三：开放生态 [待办]

- **目标**：支持 TextBundle 格式及 API 导出能力。

---

## 5. 验收准则 (Acceptance Criteria)

- [ ] 弹窗内可流畅切换主题并即时预览。
- [ ] 导出的 PNG 无字体塌陷或布局断裂。
- [ ] 二维码扫描后能准确跳转至分享页。
