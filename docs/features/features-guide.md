# JustMemo 功能模块指南 (Features Guide)

> 最后更新：2026-03-02

本文档整合了项目核心业务模块的设计细节与技术实现，为跨模块协作提供统一参考。

---

## 🏗️ 核心业务模块 (Core Modules)

### 1. 🖼️ 画廊 · 浸式相册 (Gallery)
将画廊重构为纯图片沉浸式瀑布流体验。
- **数据策略**: 复用全局 `memoCache`。通过前端正则 `/!\[.*?\]\((.*?)\)/g` 实时过滤含图片的记录，实现 SPA 切换下的秒开体验。
- **多图交互**: 卡片显示首图封面 + 数量指示器。全屏 Lightbox 支持左右切换。
- **溯源能力**: 大图视图中提供“溯源”按钮，点击后弹出 Dialog 完整复用首页 `MemoCard` 展示原始上下文。

### 2. 📍 定位与地图 (Maps)
为 Memo 提供地理空间标记能力。
- **技术栈**: Leaflet + CartoDB (底图跟随应用主题自动切换 Voyager/Dark Matter)。
- **存储格式**: 内联语法 `📍[地名](lat,lng)`，结合数据库 `locations` JSONB 字段。
- **三种视图**: 
    - 编辑器：地图选点对话框。
    - 信息流：悬浮地图小窗口预览 (HoverCard)。
    - 全页视图：展示所有标记点，支持点击后弹出原 Memo 预览。
- **交互优化**: 
    - **聚合点点击**: 采用 **一次性精确跳转 (One-shot Precise Jump)** 策略。系统通过 `flyToBounds` 算法实时计算包含该聚合组内所有点的最小边界（Bounding Box），并自动匹配最佳层级瞬时飞往目标，非固定步进（如 +2 级）缩放。配合 `padding: [50, 50]` 确保视觉留白。
    - **聚合规则**: 聚合半径设置为 `maxClusterRadius: 40`，平衡了视觉清晰度与标记密度。当地图缩放至极限层级 `18` (MaxZoom) 且点位仍重合时，启用 **蜘蛛腿 (Spiderfy)** 物理展开效果以便点击。
    - **缩放层级控制**: 
        - 最小缩放 (MinZoom)：根据容器宽度动态计算 `log2(width/256)`，确保瓦片始终铺满容器，无黑边。
        - 初始缩放：Mini 模式固定层级 `13`；全屏模式有数据时默认聚焦最新点至层级 `12`。
        - 最大缩放 (MaxZoom)：统一限制为 `18` 级，以确保底图瓦片的高可用性，避免出现灰色区域。
    - **状态指示**: 底部增加实时缩放层级指示器（带 1.5s 自动隐藏逻辑），数值采用 UI 层四舍五入处理，确保始终以整数显示层级。

### 3. 🔗 链接预览 (Link Previews)
提升外链的上下文感知。
- **机制**: 由 `MemoContent` 组件渲染。检测到标准 URL 时，异步调用特定 Action 获取 OpenGraph 元数据。
- **视觉**: 展示标题、描述与 Favicon。背景采用半透明卡片设计以保持视觉轻量。

### 4. 🗄️ 多选与批量操作 (Batch Selection)
提高内容管理效率。
- **交互逻辑**: 长按或点击“选择”按钮进入多选模式。
- **UI 变化**: 首页标题栏瞬间切换为工具栏（置顶、删除、取消）。编辑器在多选模式下通过 CSS 强制隐藏以防止误触。
- **性能**: 采用状态追踪机制，确保多选状态切换时不触发不必要的重绘（Zero-Jank）。

### 5. 📧 邮件地址处理 (Email Handling)
增强正文中的联系方式互动性。
- **自动解析**: 实时识别正文中的邮件地址并转换为 `mailto:` 链接。
- **快捷复制**: 悬浮邮件地址时显示带有平滑动画的复制按钮。
- **视觉优化**: 采用 14px 图标确保在高分屏下的亚像素渲染清晰度。

### 6. 📊 Supabase 用量监控 (Usage Monitoring)
提供项目配额透明度。
- **采集**: 通过 Management API 抓取全量指标（DB, Storage, MAU, Egress），支持 SQL 基础回退。
- **展示**: 实时变色进度条 + 模态框仪表盘。
- **位置**: 管理员设置页面。
- **详情**: 参见 [usage-monitoring.md](./usage-monitoring.md)。

### 7. 📅 时间轴与单日归档 (Timeline & Archiving)
重塑了 JustBB 核心的 Timeline 导航体验，移除了复杂的“时空穿梭”与“混合上下文翻页器”，让所有的查询和展示完全统一收敛为标准的长列表与严格的边界过滤。
- **两种浏览模式**:
    1. **严格区间模式 (Strict Bound Filter)**: 无论是在左侧的 **热力图 (Heatmap)** 点击单独的某一天，还是在右侧的 **时间轴 (Timeline)** 点击某一年/月/日，都会触发统一的 URL 参数变化（如 `/?year=2026`、`/?date=2024-05-18`）。页面会立即变为该时间段的专属归档视图。在此模式下，列表的无限滚动受限于该时间段的边界，不会发生跨月或跨年的数据溢出。
    2. **标准瀑布流模式 (Infinite Scroll)**: 在没有任何日期参数限制时（首页），默认行为为顺滑的单向（向下）全局无限瀑布流加载。**已增强底部加载微交互**：通过图标优化与强制旋转周期，确保在任何网络环境下均能提供一致且无闪烁的视觉反馈。
- **一致的数据源体验**: 
    - 不再为左侧和右侧的组件维护两套不同的“跳转”逻辑。所有组件（标签、热力图、时间轴）的点击事件全部收敛为标准的 `URL SearchParams` 操作（依赖 `router.push`）。
    - 统一的参数操作使得系统能够完美复用 `usePageDataCache` 的缓存逻辑。用户在看过某一年后，点去别的地方再点回来，可以实现瞬间重绘的 **0ms 响应**。
- **零动画负担 (Zero-Animation Strategy)**:
    - 废弃了所有基于 `teleportDate` 状态带来的全局“Teleporting”遮罩层与黑屏过渡动画，回归极简直觉的 SSR / Skeleton 列表加载反馈。

## 🛠️ 公共逻辑规范

### 内容解析 (Content Parsing)
统一由 `src/lib/contentParser.ts` 负责：
- 优先级：Code Block (```) > Location (📍) > Image > Mention (@) > Tag (#) > Link。
- 解析后的 Tokens 供 `MemoContent` 按分支渲染。

### 缓存预热 (Cache Warming)
访问任意子页面（画廊、地图）都会优先检查 `memoCache`：
- 若缓存为空，子页面会主动触发 `getMemoIndex()`。
- 该机制利用一次拉取为后续所有的 SPA 导航提供加速。
