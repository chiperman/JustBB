# JustMemo 功能模块总览

> 最后更新：2026-04-26
> 状态：总览入口

## 1. 作用

`docs/features/` 用来描述 JustMemo 的业务功能、页面能力和模块边界。

如果你要了解的是：

- 用户在某个页面能做什么
- 某个业务模块有哪些交互规则
- 某个功能和其他模块如何协同

优先从这里进入。

## 2. 阅读顺序

如果你是从用户主流程理解项目，推荐按下面顺序阅读：

1. [内容体验](./content-experience.md)
2. [多选与批量操作](./selection.md)
3. [时间轴与归档](./timeline.md)
4. [地图功能](./map.md)

## 3. 模块分组

### 浏览与导航

- [时间轴与归档](./timeline.md)
  负责日期筛选、严格区间浏览、热力图与时间轴跳转，以及“那年今日”能力。

- [地图功能](./map.md)
  负责地理坐标展示、地图标记、聚合点交互和私密 Memo 在地图中的锁定态表现。

- [画廊功能](./gallery.md)
  负责图片 Memo 的聚合浏览、全屏查看和回溯原文。

### 内容交互

- [内容体验](./content-experience.md)
  负责正文解析、链接预览、邮件地址交互和子页面缓存预热策略。

- [多选与批量操作](./selection.md)
  负责列表选择模式、批量操作入口和编辑器隐藏策略。

### 管理与运维

- [Supabase 用量监控](./usage-monitoring.md)
  负责管理员侧资源配额监控与刷新能力。

## 4. 与其他目录的边界

- 功能使用到的接口、RLS、RPC 或查询约束，请看 `docs/core/`。
- 功能涉及的动效、视觉组件规范，请看 `docs/interface/`。
- 功能开发流程、测试策略与文档维护规则，请看 `docs/guide/`。

## 5. 维护规则

- `features-guide.md` 只做总览，不再承载完整实现细节。
- 当某个功能主题超过一个屏幕且有独立规则时，应拆为单独文档。
- 如果某个模块开始拥有多个子能力，可以继续在 `docs/features/` 下横向扩展，而不是把所有内容塞回总览页。
