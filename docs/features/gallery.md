# JustMemo 画廊功能

> 最后更新：2026-04-26
> 状态：已实现

## 1. 作用

画廊页用于聚合所有包含图片的 Memo，提供比主列表更沉浸的图片浏览体验。

## 2. 功能目标

- 从普通 Memo 流中抽取图片内容
- 用更适合视觉浏览的方式展示图片
- 仍然保留回到原始 Memo 上下文的能力

## 3. 数据来源

- 数据由含图片的 Memo 构成。
- 图片识别依赖正文中的 Markdown 图片语法。
- 画廊页优先复用全局缓存，减少子页面切换时的等待。
- 画廊当前只展示公开 Memo，不显示私密 Memo，也不提供锁定占位。

## 4. 核心体验

### 4.1 图片聚合

- 卡片展示首图封面。
- 多图 Memo 会显示数量指示。
- 用户可以直接在画廊中浏览，而不必先回到主列表。

### 4.2 全屏查看

- 支持进入沉浸式大图视图。
- 大图视图支持在同一条 Memo 的多张图片之间切换。

### 4.3 回溯原文

- 大图视图中保留“查看原文”能力。
- 原文展示复用首页的 `MemoCard` 语义，而不是重新发明一套展示组件。

## 5. 和主列表的关系

- 主列表负责完整上下文。
- 画廊负责图片优先浏览。
- 两者不是两套完全独立的数据模型，但当前查询口径并不完全相同。
- 主列表可能混合公开 Memo、作者本人私密 Memo 和锁定占位；画廊当前只取公开且含图片的 Memo。

## 6. 维护边界

- 图片提取规则变化时，应同步更新正文解析相关说明。
- 如果未来增加视频、音频等富媒体能力，建议从画廊中继续拆出“媒体内容”层，而不是把所有富媒体都塞进一页。
