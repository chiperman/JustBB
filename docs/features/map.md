# JustMemo 地图功能

> 最后更新：2026-04-26
> 状态：已实现

## 1. 作用

地图页用于聚合所有带坐标的 Memo，并以空间位置而不是时间顺序进行浏览。

## 2. 数据模型

- 位置信息使用内联语法 ``📍[地名](lat,lng)`` 写入正文。
- 解析后落入数据库 `locations` JSONB 字段。
- 地图视图读取所有含 `locations` 的 Memo。

## 3. 三种使用场景

### 3.1 编辑器选点

- 在编辑器中打开地图选点对话框。
- 选择坐标后将位置写回 Memo 正文和结构化字段。

### 3.2 信息流预览

- 在主列表中可通过悬浮预览看到地图小窗。
- 该模式用于快速查看位置，不替代完整地图页。

### 3.3 全页地图

- 全页地图展示所有可定位 Memo 的标记点。
- 用户点击后可查看对应 Memo 的地图弹窗内容。

## 4. 交互规则

### 4.1 聚合点行为

- 远距离聚合点使用平滑推进方式放大至目标区域。
- 当标记坐标在当前分组口径下被视为同一位置，或已经到最大缩放层级时，不使用蜘蛛腿展开，而改为聚合列表弹窗。
- 当前“同一位置”的判定会基于坐标精度收敛，而不是要求原始浮点值逐位完全一致。

### 4.2 弹窗内容

- 单个弹窗最多展示 3 条 Memo 卡片。
- 超出部分使用内部滚动，不扩大地图整体遮挡。
- 弹窗卡片支持内容展开与折叠。
- 卡片保留“查看原文”能力，允许返回主页定位该条记录。

### 4.3 缩放控制

- 最小缩放按容器宽度动态计算，避免瓦片黑边。
- 全图最大缩放统一限制为 `18`。
- 地图底部显示实时缩放层级指示，并在短暂空闲后自动隐藏，减少视觉干扰。

## 5. 私密 Memo 规则

地图是私密 Memo 的特殊场景，规则如下：

- 只要 Memo 有坐标，就允许在地图上显示标记。
- 私密 Memo 的标记可以存在，但默认只显示锁定态。
- 作者本人点击私密标记可直接看到正文。
- 非作者点击私密标记后，必须先解锁该条 Memo，解锁后才显示具体内容。

这与主列表、搜索、标签中的权限模型保持一致：作者本人可见，非作者按条解锁。

## 6. 技术边界

- 地图底图来自 CartoDB。
- 地图交互基于 Leaflet。
- 地图页读取的数据允许包含私密 Memo 的锁定占位，因此和普通公共查询不同。

## 7. 后续扩展建议

- 如果未来增加地点筛选、城市聚合或轨迹回放，建议继续在地图文档中扩展。
- 如果未来地图能力演变成独立产品面板，可再拆分地图数据、地图交互和地图视觉三个文档。
