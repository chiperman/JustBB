# JustMemo 多选与批量操作

> 最后更新：2026-04-26
> 状态：已实现

## 1. 作用

多选模式用于提高列表管理效率，让用户可以在同一批 Memo 上执行统一操作。

## 2. 进入方式

- 点击显式的“选择”入口

进入后，列表从普通浏览模式切换到批量操作模式。

## 3. 主要行为

### 3.1 顶部工具栏切换

- 首页标题栏会切换为批量操作工具栏。
- 批量工具栏承载置顶、删除、取消等动作。

### 3.2 页面状态切换

- 多选模式下，首页标题区域会切换到批量操作语义。
- 选择状态由全局上下文维护，批量操作通过独立工具栏承载。
- 这套模式的重点是把页面从“浏览 / 发布”切换到“批量管理”，而不是额外创建一个独立页面。

### 3.3 状态管理

- 多选状态由独立上下文维护。
- 选中集合使用 ID 集合而不是列表副本。
- 批量操作只针对当前已选 Memo。

## 4. 性能要求

- 进入或退出多选模式时，不应触发整页明显抖动。
- 列表项应尽量避免不必要重绘。
- 顶部区域与批量工具栏切换时，内容区应保持稳定。

## 5. 和其他模块的关系

- 多选是主列表的操作模式，不是单独页面。
- 与私密 Memo 权限模型并不冲突，但批量动作仍然只能作用于当前用户有权限操作的 Memo。

## 6. 后续扩展建议

- 如果未来增加“批量打标签”“批量移动”“批量导出”，建议继续在本页追加规则。
- 如果批量操作种类显著增加，再单独拆出“批量动作矩阵”文档。
