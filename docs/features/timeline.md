# JustMemo 时间轴与归档

> 最后更新：2026-04-26
> 状态：已实现

## 1. 作用

时间轴模块负责按日期维度浏览 Memo，并统一左侧热力图、右侧时间轴和主列表之间的跳转行为。

## 2. 设计目标

- 用统一的 URL 参数表达所有日期筛选
- 保持主列表与归档页使用同一套数据流
- 避免旧版“时空穿梭”式的复杂状态切换

## 3. 浏览模式

### 3.1 严格区间模式

当 URL 存在日期边界参数时，例如：

- `?date=2024-05-18`
- `?year=2026`
- `?year=2026&month=03`

页面进入严格区间模式：

- 列表只展示该时间段内的 Memo
- 无限滚动受时间边界限制
- 不允许跨月或跨年溢出到其他区间

### 3.2 标准瀑布流模式

当没有日期筛选参数时：

- 首页维持全局时间倒序瀑布流
- 向下加载更多历史内容
- 底部加载交互只负责顺滑反馈，不引入额外的导航动画负担

## 4. 统一跳转模型

以下入口都收敛到同一套 URL 参数机制：

- 左侧热力图
- 右侧时间轴
- 与日期相关的页面内跳转

这样做的结果是：

- 不再维护多套“跳去某一天”的逻辑
- 可以直接复用 `PageDataCache`
- 用户返回已访问过的日期视图时可获得更快的重绘体验

## 5. 私密 Memo 规则

- 时间轴本身不负责解锁私密 Memo。
- 首页中由日期筛选驱动的主列表，私密 Memo 仍按主列表规则处理。
- 但当前独立归档查询口径只返回公开 Memo，不显示私密 Memo，也不提供锁定占位。
- 因此“时间维度浏览”在当前实现里有两种口径：主列表日期筛选是混合可见性模型，独立归档结果是公开内容模型。

## 6. 那年今日

“那年今日”属于时间维度浏览能力的一部分：

- 按当前月日回溯往年记录
- 提供独立视图，而不是继续挤在侧边栏里
- 用于提升历史内容的再次访问优先级

## 7. 维护边界

- 时间轴负责日期维度的导航和过滤。
- 统计口径、热力图聚合和私密可见性规则，应分别以 `core` 文档中的统计和安全说明为准。
