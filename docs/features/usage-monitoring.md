# JustMemo Supabase 用量监控

> 最后更新：2026-04-19
> 状态：已实现

## 1. 作用

用量监控模块用于让管理员查看当前 Supabase 项目的资源配额使用情况，降低因为免费额度或项目上限被打满而导致服务异常的风险。

## 2. 功能范围

当前监控覆盖：

- 数据库存储
- 流量相关指标
- 存储占用
- 月活相关指标
- Realtime 峰值连接

## 3. 数据来源

### 3.1 优先来源

优先通过 Supabase Management API 获取完整指标。

### 3.2 回退来源

当环境未配置管理 API Key 时，回退到基础 SQL 统计，例如：

- 数据库体积
- 用户数量

## 4. 使用方式

- 入口位于管理员设置区域
- 展示方式为弹窗仪表盘
- 支持手动刷新

## 5. 权限边界

- 只有具备管理员权限的用户才能看到入口与数据
- `SUPABASE_MANAGEMENT_API_KEY` 只能存在于服务端环境变量
- 前端浏览器不能直接调用外部管理 API

## 6. 视觉反馈

用量状态通过进度条颜色表达：

- 低风险：绿色
- 预警：黄色
- 高风险：红色

核心目标是让管理员快速判断“现在是否需要处理”，而不是让页面变成复杂的运维后台。

## 7. 后续扩展方向

- 增加阈值告警
- 增加历史趋势快照
- 如果后续管理员功能明显增多，可以考虑迁入单独的 `features/admin/` 目录
