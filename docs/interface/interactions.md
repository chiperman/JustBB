# JustMemo 交互与动画手册 (Interactions & Motion)

> 最后更新：2026-04-06 (同步滚动联动收缩与视口感知逻辑)

本文档定义了 JustMemo 的核心交互逻辑与动画规范，旨在确保全站操作的流畅性、一致性与响应感。

---

## 1. 编辑器转场策略 (Editor Transitions)

为了解决编辑器展开时的抖动，我们采用 **非对称动画** 与 **滚动联动收缩** 策略。

### 1.1 动画参数
*   **展开 (Expand)**: 使用 `ease.out` (Quintic)。避免展开初期的物理计算抖动。
    - `duration: 0.4, ease: [0.22, 1, 0.36, 1]`
*   **收缩 (Collapse)**: 使用自适应弹簧。提供极佳的物理回弹手感。
    - **设计核心参数**: `stiffness: 350, damping: 40` (此参数经过多设备调优，兼顾速度与稳定性)。

### 1.2 滚动联动与迟滞算法 (Hysteresis Logic)
为了防止短内容页面在滚动时产生“果冻效应”（即高度改变导致滚动条跳变，进而反复触发状态切换），执行以下逻辑：
- **触发阈值**: 页面向下滚动 > 100px 时触发收缩；向上滚动恢复至 < 50px 时触发展开。
- **迟滞保护**: 收缩触发时，若当前处于聚焦状态，主动调用 `editor.commands.blur()` 以确保视觉上的干净利落。
- **空间检查**: 仅在内容区总可滚动高度 > 300px 时允许动态收缩，防止在单条 Memo 页面产生布局抖动。

## 2. 内容感知交互 (Content Awareness)

### 2.1 新内容指示器 (New Memos Indicator)
当有新笔记从远端（Realtime/SWR）插入但不在当前视口内时，显示悬浮引导胶囊：
- **检测机制**: 采用 **IntersectionObserver** 监听 `lastCreatedId` 对应的 DOM 节点。
- **高性能**: 彻底摒弃传统的 `scroll` 事件轮询。仅在元素进入/离开视口时触发状态变更。
- **微任务同步**: 使用 `Promise.resolve().then()` 延迟初始化状态更新，规避 React 级联渲染产生的性能警告。

---

## 3. 全屏编辑交互 (Fullscreen Editor)

### 3.1 结构化同步 (JSON Sync)
为了彻底解决换行丢失问题，草稿同步从 Text 模式升级为 **JSON 模式**。
- 存储：`localStorage` 存储 `editor.getJSON()`。
- 恢复：自动检测并回退至 Plain Text 以保证向下兼容。

### 3.2 选区与焦点
- **页脚焦点锁定**: 点击操作按钮（私密、置顶）时 `preventDefault`，防止编辑器失焦导致非预期收缩。

---

## 4. 登录与入场动效 (Login Experience)

### 4.1 状态机设计 (Three-Stage State Machine)
登录页采用三段式状态机，通过 `LayoutContext` 的 `viewMode` 驱动：
- `HOME_FOCUS`: **全屏沉浸模式**。首页占据 100% 宽度，无边框圆角。
- `CARD_VIEW`: **中间过渡态**。首页收缩为带有 24px 圆角的卡片（停留 400ms）。
- `SPLIT_VIEW`: **双卡片对等模式**。

---

## 5. 防退化检查 (Regression Checklist)
- [x] **展开/收缩**: 整体平稳，无晃动，无“果冻效应”。
- [x] **视口感应**: 新内容指示器在滚动到目标位置后需自动消失。
- [x] **自动失焦**: 滚动触发收缩时，编辑器光标应被平滑移除。
- [x] **多选切换**: 编辑器瞬间出现/消失，内容保持静止。
