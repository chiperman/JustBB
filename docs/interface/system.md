# JustMemo 设计系统

> 最后更新：2026-04-19
> 状态：当前视觉规范基线

## 1. 作用

这份文档定义 JustMemo 的视觉系统，包括：

- 色彩
- 排版
- 圆角
- 阴影
- 毛玻璃
- 对齐与滚动规范

## 2. 视觉方向

JustMemo 当前的视觉方向可以概括为：

- 极简
- 圆润
- 克制
- 以阅读与记录为中心

它不是强装饰型界面，也不是纯工具感后台。视觉目标是“足够柔和，但不失效率”。

## 3. 配色系统

### 3.1 基础原则

- 支持浅色 / 深色主题
- 底层使用 OKLCH 色值体系
- 主界面避免高饱和噪音色

### 3.2 强调色

- 主强调色：赤陶色
- 次强调色：深橄榄色
- 状态色：使用软化后的成功 / 错误色

强调色的职责是提示与聚焦，不是大面积铺色。

## 4. 排版系统

- 正文默认使用无衬线字体
- 标题保持与正文同一视觉语言
- 等宽字体只用于数字、标签、代码块等需要对齐的场景

排版目标是稳定，而不是制造过强的品牌戏剧性。

## 5. 形状系统

### 5.1 圆角层级

| 层级 | 物理尺寸 | 语义 |
|------|------|------|
| 一级容器 | 12px | 卡片、侧边栏、对话框底座 |
| 二级容器 | 8px | 内嵌面板、下拉层 |
| 原子组件 | 6px | 按钮、输入框、菜单项、徽章 |

### 5.2 使用原则

- 交互元素默认使用 6px 圆角
- 不要在同一区域混用过多圆角尺度
- 地图等底层图形元素可以使用更小圆角

## 6. 阴影与海拔

JustMemo 使用克制的海拔体系，而不是强烈悬浮感。

| 层级 | 用途 |
|------|------|
| `shadow-none` | 导航项、基础元素 |
| `shadow-sm` | 小型独立操作按钮 |
| `shadow-md` | 卡片悬停、主操作反馈 |
| `shadow-xl` | 对话框、弹出层 |

约束：

- 阴影颜色保持中性
- 导航类元素悬停时优先用背景变化，不依赖阴影
- 编辑器聚焦态优先使用边框或 ring，而不是抬高海拔

## 7. 毛玻璃规范

以下浮层统一使用玻璃感材质：

- Dialog
- Dropdown
- Tooltip
- Command 面板
- 地图弹窗

核心参数：

- 不透明度约 88%
- 模糊度 24px
- 弱边框强化层级感

## 8. 图标与文本对齐

- 小尺寸文本优先搭配等高图标
- 14px 文本搭配 14px 图标是常用基线
- 图标库统一使用 Hugeicons

对于复制、邮件等小尺寸操作图标，需要优先保证高分屏下的清晰度。

## 9. 对齐与滚动规范

### 9.1 三层对齐模型

所有需要横向稳定对齐的区域都应尽量遵守：

1. Container
2. Constraint
3. Padding

典型组合为：

- 外层负责滚动与宽度占位
- 中层负责 `max-width` 与居中
- 内层负责 `px-4 / px-6`

### 9.2 滚动条策略

- 主容器优先使用稳定 gutter
- 不要把“隐藏滚动条”和“保留 gutter”混写成互相打架的方案
- 内容稳定性优先于表面上的极简

## 10. 地图视觉补充

地图页是系统内少数视觉风格偏“空间感”的页面，但仍需遵守主设计语言：

- 底图服务保持稳定与清晰
- 弹窗继续使用系统级玻璃材质
- 地图不会引入一套完全独立的品牌样式
