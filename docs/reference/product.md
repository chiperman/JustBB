# JustMemo 产品能力参考

> 最后更新：2026-07-14
> 状态：当前实现参考

## 1. 作用

这份文档只记录已实现的产品能力和边界。待办与方案通过 GitHub Issues 管理，不写入这里。权限、RLS、RPC、数据库和 Server Actions 的真相见 [数据与隐私参考](./privacy-and-data.md)；页面结构和缓存协同见 [架构参考](./architecture.md)。

## 2. 浏览与导航

### 时间轴与归档

时间轴负责按日期维度浏览 Memo，并统一左侧热力图、右侧时间轴和主列表之间的跳转行为。

当前 URL 参数：

- `?date=YYYY-MM-DD`
- `?year=YYYY`
- `?year=YYYY&month=MM`

有日期边界时进入严格区间模式：

- 列表只展示该时间段内的 Memo。
- 无限滚动受时间边界限制。
- 不跨月或跨年溢出到其他区间。

无日期筛选时，首页维持全局时间倒序瀑布流。

“那年今日”当前是部分实现：server action 查询逻辑已在 `src/server/actions/memos/query.ts`，独立 UI 页面尚未实现。

### 地图

地图页聚合所有带坐标的 Memo。

- 位置信息使用“图钉 + 地名 + 经纬度”的内联语法写入正文。
- 解析后落入 `locations` JSONB 字段。
- 地图交互基于 Leaflet，底图来自 CartoDB。
- 地图允许在匿名状态下也显示私密 Memo 的锁定标记（灰色锁定图钉，保留地理坐标用于展示但其正文已被完全脱敏），点击后可在弹窗或侧边进行原地口令解锁。

当前地图支持：

- 编辑器选点
- 信息流地图预览
- 全页地图
- 聚合点平滑推进
- 同位置聚合列表弹窗
- 弹窗内查看原文

### 画廊

画廊页聚合包含图片的 Memo，提供图片优先浏览体验。

- 图片识别依赖正文中的 Markdown 图片语法和图片直链解析。
- 卡片展示首图封面。
- 多图 Memo 显示数量指示。
- 大图视图支持同一条 Memo 内多图切换。
- 保留查看原文能力。

- 未解锁的私密 Memo 可显示锁定占位卡片。原图地址不会返回；验证口令后，真实图片和正文只写入当前页面内存。

## 3. 内容体验

正文统一由内容解析层处理，解析优先级为：

1. Code Block
2. Location
3. Markdown Image
4. Mention / Ref
5. Tag
6. Email
7. Raw Image URL
8. Markup Link
9. Raw Link

已实现重点：

- Markdown 图片和图片直链区分处理。
- 支持 `🔗[title](url)` 标记链接语法和显示模式变体。
- 坐标解析直接影响地图功能。
- 图片解析直接影响画廊体验。

规划中能力：

- 链接预览：异步请求外链元数据，展示标题、描述和站点标识。
- 邮件地址交互：识别邮箱、渲染 `mailto:`、提供快捷复制。
- 子页面缓存预热：访问画廊、地图等页面时复用或预热 Memo 轻量索引。

## 4. 多选与批量操作

多选模式用于在同一批 Memo 上执行统一操作。

- 通过显式“选择”入口进入。
- 首页标题栏切换为批量操作工具栏。
- 选择状态由全局上下文维护。
- 选中集合使用 ID 集合，而不是列表副本。
- 批量动作只能作用于当前用户有权限操作的 Memo。

进入或退出多选模式时，应避免整页明显抖动和不必要重绘。

## 5. 导出、备份与分享

### 备份导出

备份导出为作者提供自己的 Memo 数据备份能力。

当前导出能力分为两类：

- 设置页直接导出：通过 `DataExporter` 调用 `exportMemos`，支持 Markdown 和 JSON，一次性生成文件并下载。
- 全局分页导出：通过 `ExportContext`、`ExportConfigDialog` 和 `ExportProgressPanel` 管理分页导出，支持进度展示、暂停、继续和取消。

两类导出都必须遵守同一条权限规则：只能导出当前作者自己的 Memo。

全局分页导出路径：

1. 用户选择导出格式。
2. 前端按页拉取当前作者 Memo。
3. 进度面板展示状态。
4. 用户可以暂停、继续或取消。
5. 完成后触发浏览器下载。

设置页直接导出路径：

1. 用户点击 JSON 或 Markdown 导出。
2. Server Action 查询当前作者 Memo。
3. 浏览器直接下载生成文件。

### 单条分享与海报

单条 Memo 分享目标是生成极简排版图片。

规划或规格重点：

- 多主题排版。
- 日期、二维码、品牌标识显隐控制。
- 图片下载与剪贴板复制。
- 生成过程中保持按钮布局稳定。

私密 Memo 不允许生成对外分享图，也不允许提供对外分享能力。

## 6. 管理与运维

Supabase 用量监控用于让管理员查看资源配额使用情况。

当前覆盖：

- 数据库存储
- 流量相关指标
- 存储占用
- 月活相关指标
- Realtime 峰值连接

数据优先通过 Supabase Management API 获取；未配置 Management API Key 时，回退到基础 SQL 统计。

权限边界：

- 只有具备管理员权限的用户才能看到入口与数据。
- `SUPABASE_MANAGEMENT_API_KEY` 只能存在于服务端环境变量。
- 前端浏览器不能直接调用外部管理 API。

## 7. 功能维护原则

- 功能文档只描述当前行为和明确规划，不写历史流水账。
- 如果功能会暴露正文、标签、统计、导出或分享内容，必须先核对 [数据与隐私参考](./privacy-and-data.md)。
- 如果功能改变页面状态、URL 参数、缓存或 Provider 协同，必须同步核对 [架构参考](./architecture.md)。
- 如果功能只是 UI 表现或动效变化，优先核对 [设计系统](../interface/design.md)。
